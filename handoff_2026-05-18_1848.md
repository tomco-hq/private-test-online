# Handoff — 2026-05-18 18:48

## What this project is

`tomco-site/` is a static e-commerce site for "TomCo" (handmade goods), deployed
to GitHub Pages, using Snipcart (v3.7.1) for cart/checkout. `products.json` is the
single source of truth for the catalog; `build.py` is a static site generator that
reads it and regenerates pages.

## The task that was completed

Built a long-form sales / landing page for one product — the **Sunrise Print**
($25.00, `products.json` id `print-001`, slug `sunrise-print`). It is a
stand-alone marketing campaign page, separate from the main store. This work is
**done**. Files created/modified are listed below.

## Files that matter

- `tomco-site/offer.html` — the landing page. Stand-alone (no shared site nav),
  10 sections, `<body class="offer-page">`. Loads `/style.css` then `/offer.css`;
  `/script.js` then `/offer.js` before `</body>`.
- `tomco-site/offer.css` — landing-page-specific styles. Composes on top of
  `style.css` `:root` tokens (`--bg`, `--fg`, `--muted`, `--accent`, `--border`).
- `tomco-site/offer.js` — page-local JS: footer year, scroll reveal, sticky CTA
  bar, countdown timer.
- `tomco-site/build.py` — site generator; was extended to also manage `offer.html`.
- `tomco-site/products.json` — catalog source of truth.
- `tomco-site/sitemap.xml` — generated; now includes `/offer.html` (13 URLs).
- `.claude/launch.json` — preview server config; the relevant server is named
  `site` (Python http.server on port 8000, serving `tomco-site/`).

## Key decisions and the reasoning behind them

- **Stand-alone campaign page, no site nav.** A sales page has a single conversion
  goal; site navigation leaks attention away from the CTA. Deliberate.
- **Sells a real catalog item** (`print-001`) so the Snipcart "Add to cart" button
  works end-to-end, not a mock.
- **Separate HTML/CSS/JS files** (not inlined) — explicitly requested.
- **`offer.css` composes with `style.css` rather than redefining tokens.** Load
  order matters: `style.css` first (defines `:root` vars + dark mode), `offer.css`
  second. `offer.css` uses its own class namespace — no conflicts with site CSS.
- **`build.py` is the single source of price truth for `offer.html`.** The page's
  buy buttons and price text were originally hardcoded, which could desync from
  `products.json`. Now `build.py` regenerates them. This matters because Snipcart
  crawls each button's `data-item-url` and **rejects checkout if `data-item-price`
  mismatches the product page** — a desynced price silently breaks buying.
  - Mechanism: HTML-comment BUILD markers in `offer.html`
    (`<!-- BUILD:TAG -->` ... `<!-- /BUILD:TAG -->`). Tags: `buy-hero`,
    `buy-offer`, `buy-final`, `buy-sticky` (4 buy buttons, all `print-001`,
    different CTA wording), plus `price` and `sticky-text`.
  - `build.py` additions: `OFFER_FILE` path const, `OFFER_PRODUCT_ID = "print-001"`,
    `offer_buy_button()`, `_replace_offer_region()`, `update_offer()` (called from
    `main()` after `update_index()`), and `/offer.html` added to `STATIC_PAGES`.
  - To change the offer price: edit `products.json`, then re-run
    `C:\anaconda\python.exe build.py` from `tomco-site/`.
- **Mobile above-the-fold fix.** A `@media (max-width: 480px)` block in `offer.css`
  shrinks the hero image (360px → 160px) and tightens spacing so the primary CTA
  button is visible without scrolling at 375px and 390px widths. Verified: CTA
  bottom ~449–450px, well within a 667px viewport.
- **Countdown timer is a fixed real deadline, not a fake reset-on-load timer.**
  The original counted 24h from page load (reset every refresh — a trust risk).
  Now `offer.js` has `DEADLINE = new Date("2026-07-04T23:59:59")`. Shows the same
  value for every visitor (`Nd HH:MM:SS` format), displays "Offer ended" when
  passed. **The July 4 2026 date was set arbitrarily** at the user's instruction —
  it is a placeholder, not a real promo deadline.

## Conventions / preferences established

- Honest marketing only — no fake/resetting urgency. (Why the countdown was fixed.)
- All page copy is placeholder marketing text in the "TomCo voice," meant to be
  edited. Clearly editable, not final.
- `build.py` must remain the price authority for `offer.html`; do not re-hardcode
  prices into the page.
- Environment: Windows, PowerShell, Python at `C:\anaconda\python.exe`.

## Known environment constraints (about the preview tool, not the code)

- The Claude preview renderer is **non-painting**: `IntersectionObserver` and
  `requestAnimationFrame` callbacks do not fire, and `preview_screenshot` times
  out. This is why `offer.js` deliberately uses a **scroll-event handler with a
  `Date.now()` 80ms throttle** for scroll reveal and the sticky CTA — NOT
  IntersectionObserver/rAF. Do not "modernize" this back to IO/rAF; it would
  silently break in that environment. Verification is done via DOM inspection
  (`preview_eval`), not screenshots.
