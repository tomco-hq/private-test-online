## How the site is built

`build.py` is the static-site generator. `products.json` is the **single
source of truth** for the catalog. Running `build.py`
(`C:\anaconda\python.exe build.py` from the `tomco-site` folder):

- Writes `shop.html` (grid of all products).
- Writes one `products/<slug>.html` detail page per product.
- Rewrites marked regions of hand-built pages in place, between HTML-comment
  markers — `<!-- BUILD:TAG -->` ... `<!-- /BUILD:TAG -->`:
  - `index.html` — `BUILD:FEATURED` region (featured-products grid).
  - `home-v2.html` — `BUILD:FEATURED` region (optional trial page; updated
    with `required=False` so build.py won't error if the file is removed).
  - `offer.html` — buy buttons + price text (`BUILD:buy-hero`, `buy-offer`,
    `buy-final`, `buy-sticky`, `price`, `sticky-text`).
- Writes `sitemap.xml` and `robots.txt`.

**Re-run `build.py` after every `products.json` change.** Everything outside
the BUILD markers on hand-built pages is left untouched.

Snipcart price safety: Snipcart crawls each button's `data-item-url` and
rejects checkout if `data-item-price` mismatches the product page. build.py
generating all buy buttons from `products.json` is what prevents desync —
do not hand-write buy buttons with literal prices.

## Key decisions and the reasoning

- **Sales/offer pages stay single-product, single-CTA, standalone** — no site
  nav, no other products. A campaign page keeps one conversion goal. One
  `offer.html` per product you want to push.
- **CTA** = "call to action" (the buy buttons). A sales page repeats ONE CTA
  down the page so the visitor can act the moment they're convinced.
- **`offer.html` deliberately does NOT replicate other pages' content.** It
  links out (`/refund.html`, `/learn-more.html`) rather than duplicating.
- **The one-page home (`home-v2`) is the right direction for a 15-product
  store** — surfaces products + story + contact in one scroll, promotes the
  offer page (the current `index.html` never links to `offer.html` at all).
- **Header nav stays minimal (4 anchors).** Secondary links belong in the
  footer, not the header.
- **Products section = full inline grid; gallery section = teaser.** Products
  are the conversion surface (show all, with Add-to-cart). The gallery is
  open-ended browsing, so teaser + link-out is fine.
- **Featured teaser threshold:** inline full product grid is fine up to
  ~10–12 products; at 13–20 switch `#products` to a curated teaser (6–8 best,
  link to full shop). This is now a config change only — set `FEATURED_LIMIT`
  and flip `"featured"` flags in `products.json`; build.py renders the subset
  on the home page and all products on `shop.html`.
- **Infinite scroll: NOT recommended** for this site. Catalog too small;
  build.py emits static HTML. The only future candidate is `gallery.html`
  (lazy-loading) once it grows large. Long-scroll (finite, real footer) is
  fine and is what `home-v2` is; infinite scroll (no end) is not.
- **`home-v2.css`/`offer.css` compose with `style.css`** — they load
  `style.css` first and reuse its `:root` tokens (`--bg`, `--fg`, `--muted`,
  `--accent`, `--border`). They use their own class namespaces; no conflicts.

## Environment / technical notes

- IntersectionObserver and requestAnimationFrame **do not fire** in the
  Claude Preview environment (non-painting renderer). Both `offer.js` and
  `home-v2.js` therefore use a **scroll-event handler with an 80ms
  `Date.now()` time-throttle** for scroll-reveal and sticky-bar logic — do
  not "modernize" these back to IO/rAF.
- `preview_screenshot` times out in this environment — verify via DOM
  inspection (`preview_eval`) instead.
- The Snipcart API key in `script.js` is a **placeholder** — checkout fails
  with "Invalid public API key" console errors. These errors are EXPECTED
  and are not a bug in any page code. Only the user can supply a real key.
- Other placeholders the user must fill: real product photos (replacing
  `placeholder.svg`), product descriptions, About copy, social handles
  (`YOUR_HANDLE`, `YOUR_CHANNEL`), Formspree form id (`YOUR_FORM_ID`),
  YouTube `VIDEO_ID`, and confirming `og-image.png` exists.
